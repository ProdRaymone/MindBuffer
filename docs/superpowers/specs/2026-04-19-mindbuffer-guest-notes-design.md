# MindBuffer Guest Mode Notes Polish Design

日期：2026-04-19

## 背景

MindBuffer 已完成 `local-first` 改造，未登录用户现在可以直接进入首页并在本地保存记录。功能方向正确，但 guest 模式当前仍然沿用了较多“工具面板”式的视觉和交互表达，和用户想要的“像 iPhone 原生备忘录一样，打开就能快速记一条”的感受还有距离。

本轮目标不是重做产品结构，而是在不破坏 authenticated 模式的前提下，只对 guest 模式做一层轻量但体感明显的 UI/交互打磨。

## 目标

- 让 guest 模式更像“安静的本地备忘录”，而不是“需要先理解功能的工具台”
- 强化“打开就写”的第一感受
- 保持本地暂存与后续登录同步能力不变
- 尽量把改动控制在 `components/MindBuffer.tsx` 的渲染层和少量样式逻辑内
- 保持 authenticated 模式的结构、能力与主流程稳定

## 非目标

- 不重做数据层，不调整 `local-first` 的存储或同步机制
- 不改变 authenticated 模式的主要信息架构
- 不新增复杂动画、手势系统或新的 API 依赖
- 不在本轮解决 Claude bridge 的稳定性问题

## 采用方向

采用 `Quiet Capture` 方向。

guest 模式应被塑造成“一个安静的本地笔记本”：

- 顶部只表达最关键的信息：当前内容安全保存在本机，可稍后登录同步
- 输入区成为绝对主角，像一张正在书写的便签
- 历史记录以文本优先的笔记流形式呈现，时间、同步状态和工具入口退居次要层
- guest 不可用的云端能力尽量不占据首屏，而不是展示后再通过 toast 告知不可用

## 方案概览

### 1. 顶部区域

guest 模式下简化顶部操作区：

- 保留 `搜索`
- 保留 `登录同步`
- 不在首层展示 `Digest 历史`、`AI 整理`、`导出`
- 菜单入口保留，但语义更偏“账户/同步”，不强调工具集合

状态栏由多个 badge 改为一条柔和 banner：

- 主信息：内容保存在本机
- 次信息：登录后可同步到云端
- CTA：`登录同步`

authenticated 模式维持当前的完整工具入口与状态结构。

### 2. 筛选与首屏密度

guest 模式下将筛选条改为默认折叠：

- 首屏不直接展示整条分类过滤和媒体过滤 chips
- 在列表区域上方提供一个轻量的 `筛选` 入口
- 用户主动点开后，再展示现有分类过滤和媒体过滤内容
- 默认折叠状态下不影响“打开就写”的首屏心智

authenticated 模式保持当前筛选结构。

### 3. 空状态

guest 空状态改为更像“第一张空白便签”的左对齐引导，而不是居中的系统提示：

- 文案强调“想到什么先写下来”
- 弱化系统说明感，强化立即开始记录的暗示

### 4. 输入区

guest 输入区改造成“便签卡片”式写作区域：

- 提高 textarea 最小高度
- 降低工具栏感，提升书写面的占比
- 占位文案改成移动端语气，不再出现 `Cmd/Ctrl + Enter`
- 发送按钮增大到更接近移动端可点击尺寸

guest 模式下隐藏图片上传按钮与 `prompt()` 式加链接按钮：

- 图片上传本来就依赖登录，首层不应展示为主要动作
- 链接输入改为鼓励直接粘贴到正文，继续复用现有 URL 自动抽取能力

authenticated 模式继续保留图片上传和链接入口。

### 5. 列表表现

guest 列表改为 text-first 的笔记流：

- 时间从左侧固定列后移到内容下方或右下角
- 分类符号保留，但弱化为辅助信息
- `local-only` 不在每条记录上高亮展示
- `syncing` 与 `failed` 仍然保留可见反馈，因为它们属于异常或过渡状态

authenticated 模式继续保留当前更结构化的记录流表达。

### 6. 移动端操作可达性

当前 `编辑/删除` 依赖 `group-hover`，在 iPhone/PWA 上不自然。本轮改为：

- guest 模式下移动端默认可见操作按钮
- 桌面端可以继续保留 hover 渐显

这项改动优先解决移动端“看得到内容但不容易操作”的问题。

## 组件与边界

主要在 `components/MindBuffer.tsx` 中通过 `sessionMode === 'guest'` 增加明确的 guest 渲染分支。

建议的边界：

- 在顶层渲染区区分 guest 与 authenticated 的 header / status / filter 呈现
- 在 composer 区区分 guest 与 authenticated 的动作按钮集合
- 在 `EntryRow` 中增加 guest 视觉分支，用于调整时间、状态与操作按钮显隐

不需要拆新数据模块；如有必要，可增加少量小型展示辅助函数，但不做结构性重构。

## 数据流与行为

本轮不改动以下行为：

- guest 写入本地草稿
- authenticated 发送时先乐观插入，再异步同步到 Supabase
- 登录后同步本地记录
- 搜索、编辑、删除等现有基础行为

本轮只改变“如何呈现这些状态”，不改变“这些状态如何产生”。

## 错误处理

- guest 模式下避免把“不可用动作”暴露在首层，减少无意义 toast
- 仍保留真正需要反馈的失败状态，例如同步失败
- 若 guest 下触发了不应出现的上传路径，继续沿用现有保护逻辑，作为兜底

## 测试与验证

实现后至少验证：

- guest 首页可直接进入
- guest 新建文字记录仍写入本地并立即出现在列表中
- guest 粘贴链接后仍能通过正文 URL 抽取生成链接附件
- authenticated 模式的图片上传、链接添加、AI 整理、导出、Digest 历史不受影响
- 本地记录登录后仍可同步
- 移动端宽度下输入区、列表与操作按钮可正常使用

命令验证继续使用：

- `npm run verify:local-first`
- `npm run type-check`
- `npm run lint`
- `npm run build`

## 实施范围

预期主要修改文件：

- `components/MindBuffer.tsx`

如需少量文案或类型补充，可涉及：

- `lib/types.ts`

但本轮不预期对数据层进行再设计。
